# 실행 결과 리포트 형식 — Task당

Phase 11 통합 테스트 **Task당 실행 결과 리포트**에 포함할 항목과 파일 명명 규칙입니다.

---

## 1. 포함 항목

| 항목 | 필수 | 설명 |
|------|------|------|
| **코드 오류** | 있으면 기재 | 테스트 중 발생한 오류. **재현 조건**, **로그/메시지**, **스택 트레이스**(가능 시). |
| **미해결 이슈** | 있으면 기재 | 테스트 시점에서 미해결인 이슈. **설명**, **우선순위**(상/중/하), **영향 범위**. |
| **해결된 이슈** | 있으면 기재 | 테스트 중 발견 후 해결한 이슈. **해결 방법**, **커밋/PR/이슈 참조**. |
| **시나리오 결과 요약** | 필수 | 시나리오 ID별 통과/실패 개수, 비고. |

---

## 2. 저장 위치

- **디렉토리**: `docs/devtest/reports/` 또는 `docs/phases/phase-11-4/`.
- **Task당 1파일**: Phase 11의 각 Task(11-1-1, 11-1-2, 11-1-3, 11-2-1 ~ 11-2-5, 11-3-1 ~ 11-3-4)당 1개 리포트 파일.

---

## 3. 파일 명명 규칙

- **형식**: `phase11-4-2-task-11-X-Y-execution-report.md`
- **예**: `phase11-4-2-task-11-1-1-execution-report.md`, `phase11-4-2-task-11-2-1-execution-report.md`

---
