## 1. 개요

### 1.1 배경

Personal AI Brain Ver3는 지식 관리 시스템의 핵심인 **청크(Chunk) 생성-승인-관리** 프로세스를 3개의 독립된 메뉴로 분리하여 제공하고 있습니다. 이 보고서는 다양한 연구·개발 조직의 리더급 전문가 10명의 관점에서 현재 시스템을 분석하고, 통합 최적화 전략을 도출하기 위해 작성되었습니다.

### 1.2 분석 목적

- 현재 3개 메뉴의 **기능적 중복 및 연계성** 파악
- 각 조직 유형별 **사용 불편점 및 개선 요구사항** 도출
- **통합 시나리오**와 **단계별 업그레이드 전략** 제시
- **국가 공공연구기관·대학·스타트업·관공서** 등 다양한 조직 특성 반영

### 1.3 분석 범위

| 메뉴          | 경로                  | 주요 기능                                 |
| ------------- | --------------------- | ----------------------------------------- |
| **청크 생성** | `/admin/chunk-create` | 파일 선택 → 분할 → 등록 (3단계 프로세스)  |
| **청크 승인** | `/admin/approval`     | Draft 청크 승인/거절, AI 추천 라벨 적용   |
| **청크 관리** | `/admin/chunk-labels` | 청크 검색, 라벨 추가/제거, AI 키워드 추천 |

## 2. 현재 시스템 구조 분석

### 2.1 청크 생성 (`/admin/chunk-create`)

**프로세스**: 3단계 마법사 형식

1. **1단계 - 파일 선택**
   - 좌측: 스토리지 파일 목록 (brain/, docs/)
   - 우측: 선택된 문서 내용 + 분할 방식 선택 (문단/한 줄씩/헤딩)

2. **2단계 - 분할·선택**
   - 좌측: 선택/병합 컨트롤
   - 우측: 분할된 청크 목록 (체크박스 선택)

3. **3단계 - 등록**
   - 좌측: 등록 방식 (새로운 청크 / 기존 문서에 추가)
   - 우측: 문서 정보 입력 (경로, 이름, 프로젝트)

**특징**:

- 단계별 UI 전환 (탭 기반)
- 복잡한 병합·분할 기능 제공
- 즉시 등록되지 않고 Draft 상태로 생성

### 2.2 청크 승인 (`/admin/approval`)

**레이아웃**: 좌우 분할

- **좌측**: 청크 목록 + 상태 필터 (대기중/승인됨/거절됨) + 페이징 + "전체 승인" 버튼
- **우측**: 선택된 청크 상세 (내용, AI 추천 라벨, AI 관계 추천)

**주요 기능**:

- 상태별 필터링
- 개별 승인/거절
- 전체 승인 (현재 페이지)
- AI 추천 라벨 즉시 적용

**특징**:

- 청크 생성 후 필수 단계
- 승인되어야만 검색/RAG에 노출
- AI 추천을 활용한 반자동 워크플로우

### 2.3 청크 관리 (`/admin/chunk-labels`)

**레이아웃**: 좌우 분할

- **좌측**: 청크 목록 + 검색 + 페이징
- **우측**:
  - 라벨 선택 영역 (분류별 필터 + 라벨 목록 + 선택 도구)
  - 청크내 연결된 라벨
  - 청크 내용 + AI 키워드·라벨 추천

**주요 기능**:

- 청크 검색
- 라벨 추가/제거
- AI 키워드·라벨 추천
- 분류별 라벨 필터링 (프로젝트 단계/역할/도메인/중요도/키워드)

**특징**:

- 승인 이후 지속적 관리 도구
- 라벨 기반 지식 구조화
- AI 추천 재실행 가능

### 2.4 프로세스 흐름도 (현재)

```
사용자 업로드(파일)
    ↓
[청크 생성] (/admin/chunk-create)
    ├─ 1단계: 파일 선택
    ├─ 2단계: 분할·병합·선택
    └─ 3단계: 등록 → Draft 상태 저장
         ↓
[청크 승인] (/admin/approval)
    ├─ 목록 확인 (필터링)
    ├─ AI 추천 확인
    └─ 승인/거절 → Approved/Rejected
         ↓
[청크 관리] (/admin/chunk-labels)
    ├─ 청크 검색
    ├─ 라벨 추가/제거
    └─ AI 키워드 추천 재실행
```

### 2.5 현 구조의 강점

| 강점                 | 설명                            |
| -------------------- | ------------------------------- |
| **명확한 책임 분리** | 생성/승인/관리가 UI상 독립적    |
| **단계별 집중**      | 각 단계에서 필요한 기능만 노출  |
| **AI 추천 활용**     | 승인·관리 모두에서 AI 활용 가능 |
| **승인 워크플로우**  | Draft → Approved 품질 관리      |

### 2.6 현 구조의 약점

| 약점                | 영향                                    |
| ------------------- | --------------------------------------- |
| **중복 이동**       | 생성 → 승인 → 관리로 3번 메뉴 이동 필요 |
| **컨텍스트 단절**   | 각 메뉴에서 다른 청크 목록/상세 UI      |
| **AI 추천 중복**    | 승인·관리 모두 라벨 추천 기능 존재      |
| **학습 곡선**       | 3개 메뉴 각각의 UI 패턴 학습 필요       |
| **워크플로우 강제** | 생성 직후 승인 불가 (메뉴 이동 필수)    |

## 4. 페르소나별 분석 리뷰

---

### 4.1 에너지연구원 책임연구원 (김에너지 박사)

**조직**: 국가 에너지연구원 (신재생에너지 연구본부)
**경력**: 15년 (태양광·풍력 시뮬레이션 전문)
**팀 규모**: 12명 (연구원 8명, 박사후연구원 4명)
**주요 업무**: 대규모 실험 데이터 분석, 연구 보고서 작성, 국제 공동 연구

---

#### 4.1.1 현재 기능 사용 경험

**청크 생성 사용 시나리오**:

- 월 1회, 대형 프로젝트 보고서 (100~200페이지) 업로드
- 헤딩 기준 분할 후 장(Chapter) 단위로 청크 병합
- 프로젝트별 폴더 구조 유지 필요

**느낀 점**:

> "3단계 프로세스는 체계적이지만, 200페이지 보고서를 한 번에 처리하려면 2단계에서 체크박스 200개를 확인해야 합니다. '헤딩 레벨 2로 자동 병합' 같은 프리셋이 있으면 좋겠습니다."

**청크 승인 사용 시나리오**:

- 생성 직후 바로 승인 페이지로 이동
- 대부분 전체 승인 버튼 사용 (개별 확인 시간 부족)
- AI 추천 라벨은 거의 확인 안 함 (시간 소요)

**느낀 점**:

> "전체 승인은 편하지만, AI 추천을 제대로 활용 못하고 있습니다. 승인 전 일괄 라벨 적용 옵션이 있으면 나중에 검색이 쉬울 것 같습니다."

**청크 관리 사용 시나리오**:

- 분기 1회, 프로젝트 종료 시 청크 재분류
- 키워드 라벨 추가 (태양광, 풍력, 에너지저장장치 등)
- 여러 청크에 동일 라벨 추가가 번거로움

**느낀 점**:

> "청크 관리에서 '여러 청크 선택 → 라벨 일괄 추가'가 안 됩니다. 체크박스로 10개 선택 후 한 번에 라벨을 추가할 수 있으면 작업 시간이 10배 줄어듭니다."

---

#### 4.1.2 사용 불편점

| 불편점                                | 심각도 | 영향                                       |
| ------------------------------------- | ------ | ------------------------------------------ |
| **대용량 문서 분할 시 체크박스 과다** | 상     | 2단계에서 200개 항목 수동 확인 (20분 소요) |
| **전체 승인 시 AI 추천 무시**         | 중     | 나중에 검색 품질 저하                      |
| **청크 관리에서 다중 선택 불가**      | 상     | 10개 청크에 라벨 추가 시 10번 반복         |
| **3개 메뉴 이동 시 컨텍스트 손실**    | 중     | 어떤 청크를 처리 중이었는지 기억 안 남     |
| **프리셋 기능 부재**                  | 중     | 매번 동일한 분할 방식 수동 설정            |

---

#### 4.1.3 변경 요구사항

1. **자동 병합 프리셋**
   - "헤딩 레벨 2 자동 병합", "100줄 단위 자동 병합" 등 템플릿 제공
   - 프로젝트별 프리셋 저장 기능

2. **일괄 라벨 적용**
   - 승인 전 "전체 청크에 [신재생에너지] 라벨 적용" 옵션
   - 청크 관리에서 다중 선택 (체크박스) + 라벨 일괄 추가

3. **워크플로우 단축**
   - 생성 → 승인을 한 화면에서 처리하는 "빠른 승인" 모드
   - 또는 생성 3단계에서 "등록과 동시에 승인" 옵션

---

#### 4.1.4 업그레이드 시나리오 제안

**옵션 A: 통합 워크플로우 (권장)**

```
[청크 관리 통합] 메뉴
  ├─ 탭1: 생성 (기존 3단계)
  ├─ 탭2: 승인 대기 (승인/AI 추천)
  └─ 탭3: 관리 (라벨/검색)

등록 후 자동으로 탭2로 이동
승인 후 탭3에서 바로 확인 가능
```

**옵션 B: 프로세스 간소화**

- 생성 2단계에서 "AI 추천 라벨 미리보기" 제공
- 3단계에서 "등록 + 승인 + 라벨 적용" 한 번에 처리
- 전문가 모드 (3단계 유지) vs 빠른 모드 (1단계만) 선택 가능

---

#### 4.1.5 기대 효과 (국가 연구기관 관점)

| 개선 항목                    | 현재 소요 시간 | 개선 후 예상  | 절감 효과    |
| ---------------------------- | -------------- | ------------- | ------------ |
| 청크 생성 (200페이지 보고서) | 30분           | 10분          | **67% 감소** |
| 승인 + 라벨 적용             | 15분 (수동)    | 2분 (일괄)    | **87% 감소** |

## 6. 통합 UI/UX 최적화 전략

### 6.1 전략적 방향

#### 6.1.1 목표

1. **워크플로우 통합**: 생성-승인-관리를 한 화면에서 처리
2. **자동화 극대화**: AI 활용하여 수동 작업 90% 감소
3. **조직 맞춤형**: 연구기관/스타트업/관공서 등 특성 반영
4. **협업 강화**: 다단계 승인, 역할 기반 권한, 팀 워크스페이스

#### 6.1.2 핵심 원칙

- **Progressive Disclosure**: 초보자는 간단 모드, 전문가는 고급 기능
- **Context Preservation**: 단계 간 이동 시 상태 유지
- **Feedback Loop**: 사용자 행동에 즉각 반응 (로딩, 성공, 실패)
- **Accessibility**: 한글 우선, 키보드 단축키, 스크린 리더 지원

```
│                                        │
```

- **"상세 설정"** 클릭 시:

```
├────────────────────────────────────────┤

│ │ "프로젝트..." │ [프로젝트]    │       │
│ │              │ [AI]         │       │

└────────────────────────────────────────┘
```

  - 현재 페이지 청크 전체 승인

| 창업랩        | "사업계획서"    | 버전 관리 활성화, 공유 링크 생성        |
| 스타트업      | "API 문서"      | Markdown 우선, Webhook 연동             |
| 중소기업      | "정부 과제"     | 전자동 모드, HWP 내보내기               |
| 관공서        | "공문서"        | 결재선 입력, HWP 표준 서식              |

**설정 방법**:


### 7.3 최종 제안: 하이브리드 접근

**모든 조직을 만족시키는 단일 솔루션은 없다.** 따라서 다음과 같이 **3단계 모드 선택**을 제공하는 것을 권장한다:

```
설정 > 워크플로우 모드 선택

[ ] 모드 1: 간단 모드 (전자동)
    → 대상: 중소기업, 지방대, 초보자
    → 파일 업로드 → AI가 모든 것 자동 처리

[v] 모드 2: 표준 모드 (반자동, 기본값)
    → 대상: 대부분의 연구기관, 스타트업
    → 탭 기반 통합 + 빠른 승인

[ ] 모드 3: 전문가 모드 (수동)
    → 대상: 보안 중심 기관, 고급 사용자
    → 기존 3단계 프로세스 + 세밀한 제어
```

이를 통해 **사용자는 자신의 수준과 조직 특성에 맞는 모드를 선택**할 수 있고, **시스템은 단계적으로 진화**할 수 있다.
