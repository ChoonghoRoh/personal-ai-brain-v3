## 📋 요약 (1페이지)

### Gap 개요

현재 상태와 목표 상태를 비교한 결과, **핵심 기능은 90% 이상 완료**되었으며, 남은 Gap은 주로 **UI 개선**과 **워크플로우 완성**에 집중되어 있습니다.

## 🟢 Low 우선순위 Gap

### Gap 3: HWP 파일 지원

**현재 상태:**

- ✅ 기본 구조만 있음 (file_parser_service.py에 HWP 파서 스켈레톤)
- ❌ 실제 HWP 파싱 로직 미구현

**목표 상태:**

- HWP 파일을 Markdown으로 변환
- 임베딩 및 저장 지원

**해결 방안:**

1. HWP 파싱 라이브러리 조사 (pyhwp, olefile 등)
2. HWP 파서 구현
3. embed_and_store.py에 HWP 지원 추가
4. 테스트 및 검증

**예상 소요 시간:** 2-3일

**참고:** HWP는 한국 특화 형식이며, 라이브러리 의존성이 높을 수 있음.

---

### Gap 4: 통계 및 분석 대시보드

**현재 상태:**

- ✅ 기본 대시보드 있음 (`/dashboard`)
- ✅ 시스템 상태 표시 (Qdrant, PostgreSQL 연결 상태)
- ❌ 상세 통계 및 분석 기능 없음

**목표 상태:**

- 문서 통계 (총 문서 수, 타입별 분포, 크기 분포)
- 지식 통계 (총 청크 수, 라벨 분포, 관계 그래프)
- 활동 통계 (최근 업데이트, 검색 빈도, Reasoning 실행 횟수)
- 시각화 (차트, 그래프)

**해결 방안:**

1. 통계 API 엔드포인트 추가 (`/api/system/stats`)
2. 대시보드에 통계 섹션 추가
3. 차트 라이브러리 통합 (Chart.js 등)
4. 실시간 업데이트 기능

**예상 소요 시간:** 2-3일
