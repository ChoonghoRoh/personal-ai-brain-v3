# 개발자 관점 — 테스트 수행 가이드

**역할**: 버그·에러·API/네트워크·성능·엣지케이스 검증

---

## 1. 테스트 중 확인할 것

- **콘솔 에러**: 브라우저 개발자 도구(F12) → Console에 JavaScript 에러·경고가 없는지
- **API 응답/상태코드**: Network 탭에서 API 요청이 실패(4xx/5xx)하지 않는지, 응답 본문이 기대와 맞는지
- **로딩/타임아웃**: 버튼 클릭 후 로딩 표시·완료 시점이 적절한지, 과도한 대기·타임아웃 없는지
- **빈 값·잘못된 입력 시 동작**: 필수 필드 비움, 잘못된 형식 입력 시 에러 메시지·재시도 가능 여부
- **재현 단계**: 버그 발견 시 “어떤 화면에서 무엇을 했을 때 어떤 결과가 나왔는지” 단계로 기록

---

## 2. 기록 형식

| 항목           | 내용                                                 |
| -------------- | ---------------------------------------------------- |
| 시나리오 ID    | 체크리스트 번호 (예: W4.2)                           |
| 관점           | 개발자                                               |
| 재현 단계      | 1) 접속 URL 2) 수행한 조작 3) 예상 결과 4) 실제 결과 |
| 예상/실제      | 기대한 동작 vs 실제 동작                             |
| 로그/에러 요약 | 가능 시 콘솔·Network 요약 (선택)                     |

---

## 3. 실행용 프롬프트 (복사하여 사용)

**프롬프트 A — 체크리스트 수행 안내**

```
나는 개발자 관점으로 아래 웹 사용자 체크리스트를 수행합니다.
- 각 시나리오를 브라우저에서 실행하면서 다음을 확인합니다:
  1) 콘솔에 JavaScript 에러/경고가 없는지
  2) API 요청이 성공(2xx)하는지, 실패 시 상태코드·응답
  3) 로딩/완료 표시·타임아웃이 적절한지
  4) 빈 값·잘못된 입력 시 에러 처리·안내가 있는지
- 버그 발견 시 "시나리오 ID, 관점(개발자), 재현 단계, 예상/실제, 로그 요약" 형식으로 기록합니다.
```

**프롬프트 B — 결과 정리용**

```
개발자 관점 테스트 결과를 정리합니다.
- 시나리오별로: 버그·에러·API/로딩 이슈가 있으면 재현 단계와 예상/실제를 한 줄씩 요약해 주세요.
```
