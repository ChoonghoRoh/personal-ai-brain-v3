# 꼼꼼한 기획자 관점 — 테스트 수행 가이드

**역할**: 요구사항·시나리오·플로우 검증, 누락·모순·불명확한 안내 포착

---

## 1. 테스트 중 확인할 것

- **시나리오와 기획서/요구사항 일치 여부**: 체크리스트 시나리오가 실제 요구사항·기획 의도대로 동작하는지
- **단계별 플로우 자연스러움**: 메뉴 진입 → 입력/선택 → 실행 → 결과 확인 흐름이 논리적인지, 불필요한 단계나 빠진 단계는 없는지
- **메시지·라벨·안내 문구 명확성**: 버튼/필드 라벨, 도움말, 에러/성공 메시지가 사용자에게 명확한지
- **예외/에러 시 사용자 안내**: 입력 오류·API 실패·빈 결과 등에서 사용자가 다음에 무엇을 해야 하는지 안내가 충분한지

---

## 2. 기록 형식

| 항목        | 내용                                           |
| ----------- | ---------------------------------------------- |
| 시나리오 ID | 체크리스트 번호 (예: W4.2)                     |
| 관점        | 기획자                                         |
| 발견 내용   | 요구사항/플로우 관점에서의 이슈 또는 양호 사항 |
| 제안        | 요구사항·문서·안내 문구 개선 제안 (선택)       |

---

## 3. 실행용 프롬프트 (복사하여 사용)

**프롬프트 A — 체크리스트 수행 안내**

```
나는 꼼꼼한 기획자 관점으로 아래 웹 사용자 체크리스트를 수행합니다.
- 각 시나리오를 일반 웹 사용자처럼 브라우저에서 실행합니다.
- 요구사항·플로우·안내 문구 관점에서 다음을 확인하고 기록합니다:
  1) 시나리오가 기획/요구사항과 일치하는지
  2) 단계별 플로우가 자연스러운지
  3) 메시지·라벨·안내가 명확한지
  4) 예외/에러 시 사용자 안내가 충분한지
- 발견한 내용은 "시나리오 ID, 관점(기획자), 발견 내용, 제안" 형식으로 기록합니다.
```

**프롬프트 B — 결과 정리용**

```
기획자 관점 테스트 결과를 정리합니다.
- 시나리오별로: 요구사항/플로우/안내 관점에서의 이슈·양호 사항·개선 제안을 한 줄씩 요약해 주세요.
```
