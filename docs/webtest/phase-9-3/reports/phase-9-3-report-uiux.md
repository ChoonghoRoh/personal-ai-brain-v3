## 2. 요약

- **버튼/링크 식별**: 주요 CTA(검색, 질문하기, Reasoning 실행, 전체 승인 등)가 버튼 또는 링크로 구분되어 있음.
- **레이아웃·일관성**: 대시보드·검색·Ask·Reasoning·지식·관리 페이지가 container·panel 구조로 일관됨.
- **접근성**: 포커스 표시, 키보드 조작, 라벨/placeholder는 코드 상으로 존재. aria·role 등은 페이지별로 추가 점검 권장.
- **오류/안내 메시지**: #error-message, #success-message 등이 여러 페이지에 있음. 노출 조건·가독성(색상·위치)을 실제 에러 시나리오에서 확인 권장.
