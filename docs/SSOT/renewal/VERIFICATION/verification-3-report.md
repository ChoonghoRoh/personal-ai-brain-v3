## 3. 3차 리뉴얼 주요 변경

### 3.1 작업지시 분리 (GUIDES/)

**변경 내용**:
- ROLES/*.md의 § 4~5 작업지시 부분을 GUIDES/*.md로 분리
- ROLES/*.md는 역할 정의 + 코드 규칙만 포함 (읽기 분량 43% 축약)
- GUIDES/*.md는 Task/검증/테스트 시작 시에만 참조

**효과**:
- 역할별 필수 읽기 분량 대폭 감소:
  - Backend Dev: 550줄 → 400줄 (-27%)
  - Frontend Dev: 550줄 → 400줄 (-27%)
  - Verifier: 800줄 → 600줄 (-25%)
  - Tester: 450줄 → 350줄 (-22%)
- 작업지시는 필요시에만 읽기 (선택적)

**파일 구조**:
```

│   ├── backend-dev.md (73줄, 작업지시 제거)
```

## 5. 장점 분석

### 5.1 읽기 분량 최적화

| 역할 | 1차 | 2차 | 3차 | 3차 개선 |
|------|-----|-----|-----|----------|
| **Backend Dev** | 550줄 | 550줄 | **400줄** | **-27%** |
| **Frontend Dev** | 550줄 | 550줄 | **400줄** | **-27%** |
| **Verifier** | 800줄 | 800줄 | **600줄** | **-25%** |
| **Tester** | 450줄 | 450줄 | **350줄** | **-22%** |

**효과**: 세션 시작 시 필수 읽기 분량이 **평균 25% 감소**

### 5.2 선택적 읽기 전략

- **역할 정의 + 코드 규칙**: 세션 시작 시 항상 읽기 (ROLES/*.md)
- **작업지시**: Task 시작 시에만 읽기 (GUIDES/*-work-guide.md)

**장점**:
- 세션 초기 로딩 시간 단축
- 필요한 정보만 선택적으로 읽기 가능
- 작업지시는 Task 할당 직후 1회만 참조하면 됨

### 5.3 유지보수성 향상

- 작업지시 변경 시 GUIDES/*.md만 수정 (ROLES/*.md 영향 없음)
- ROLES/*.md는 역할 정의 중심 (안정성 향상)
- GUIDES/*.md는 프로세스 중심 (변경 빈도 높음)
