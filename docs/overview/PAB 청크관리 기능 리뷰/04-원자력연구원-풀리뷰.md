### 4.4 원자력연구원 선임연구원 (정원자 박사)

**조직**: 국가 원자력연구원 (원자로 안전성 연구부)  
**경력**: 12년 (원자로 열유동 시뮬레이션)  
**팀 규모**: 6명 (선임 2명, 연구원 4명)  
**주요 업무**: 안전성 평가 보고서, 규제 기관 제출 자료, 기밀 데이터 관리

#### 4.4.1 현재 기능 사용 경험

**청크 생성**: 분기 1회 안전성 평가 보고서 (150~200페이지), 기밀 등급(일반/대외비/비밀) 구분 필수, 일부 청크는 규제 기관만 열람.  
**느낀 점**: "청크에 기밀 등급을 설정할 수 없습니다. 현재는 파일명에 '[대외비]'를 붙이는데, 시스템이 자동으로 접근 제어를 해줘야 합니다."

**청크 승인**: 선임 2명 교차 검토, 승인 시 전자서명·승인 사유 필수, 거절 시 시정 조치 요구사항 명시.  
**느낀 점**: "승인 버튼만 있고, 승인 사유나 전자서명 기능이 없습니다. 규제 기관 감사 시 '누가 책임지고 승인했는지' 입증할 수 없습니다."

**청크 관리**: 기밀 청크는 검색 결과에서 제외, 접근 로그(누가·언제 열람) 필수.  
**느낀 점**: "청크 관리에서 모든 청크가 노출됩니다. 기밀 청크는 권한 있는 사람만 볼 수 있어야 하고, 열람 시 로그를 남겨야 감사 추적이 가능합니다."

#### 4.4.2 사용 불편점

기밀 등급 설정 불가(최상), 접근 제어 부재(최상), 전자서명 미지원(상), 접근 로그 부재(상).

#### 4.4.3 변경 요구사항

1. 기밀 등급·접근 제어(일반/대외비/비밀/1급비밀, RBAC, 검색·RAG 필터링).  
2. 전자서명·승인 사유(PKI 연동, 10자 이상 필수).  
3. 접근 로그(열람·수정·삭제 시 사용자·시간·IP, 관리자 조회·내보내기).  
4. 보안 강화 모드(다운로드 금지, 워터마크).

#### 4.4.4 업그레이드 시나리오 제안

**옵션 A**: 생성(기밀 등급·접근 권한) → 승인(전자서명 필수) → 관리(접근 로그).  
**옵션 B**: 원자력안전법 준수 템플릿, 규제 기관 제출용 자동 분류, 감사 보고서 자동 생성.

#### 4.4.5 기대 효과 (원자력연구원 관점)

기밀 유출 방지 접근 제어 100%, 규제 감사 전자서명+이력 완벽 보존, 내부자 위협 접근 로그 실시간 추적. 원자력안전법 제104조, IAEA, 산업기술보호법 대응.
