## 현재 기능 사용 경험

- **청크 생성**: 학생별 논문 초고 주 5~10건, 섹션별 분할·학생명 태깅.  
  **느낀 점**: "관리자만 생성 접근. 학생별 개인 공간 주고 교수는 승인만 하면 효율적."
- **청크 승인**: 주 1회 학생 청크 일괄 검토, AI 추천으로 주제 분류.  
  **느낀 점**: "'머신러닝'을 machine learning·기계학습 두 라벨로 중복 추천. 라벨 정규화 필요."
- **청크 관리**: 논문 작성 시 과거 실험 검색, Jupyter 자동 임포트 희망.  
  **느낀 점**: "API 연동 약함. Jupyter Extension으로 참고 청크 5개 자동 삽입하고 싶다."

## 사용 불편점

| 불편점 | 심각도 | 영향 |
|--------|--------|------|

## 변경 요구사항

1. 학생별 개인 공간(Workspace), 교수는 전체 승인·조회  
2. 라벨 정규화·동의어 관리 (머신러닝/ML/기계학습 통합, Thesaurus UI)  
3. API·플러그인 (REST 문서, Jupyter·Notion 예제)  
4. 논문 인용 형식 (BibTeX, APA, IEEE 내보내기)
