# 4.4 원자력연구원 선임연구원 (정원자 박사) — 인터뷰 요약

**풀 리뷰**: [04-원자력연구원-풀리뷰.md](04-원자력연구원-풀리뷰.md) · **원본**: [260221-1802-청크관리-통합-최적화-전략-10대-전문가-분석.md §4.4](../../planning/260221-1802-청크관리-통합-최적화-전략-10대-전문가-분석.md) (보존 문서)

## 프로필

| 항목 | 내용 |
|------|------|
| 조직 | 국가 원자력연구원 (원자로 안전성 연구부) |
| 경력 | 12년 (원자로 열유동 시뮬레이션) |
| 팀 규모 | 6명 (선임 2명, 연구원 4명) |
| 주요 업무 | 안전성 평가 보고서, 규제 기관 제출, 기밀 데이터 관리 |

---

## 현재 기능 사용 경험

- **청크 생성**: 분기 1회 150~200페이지, 기밀 등급(일반/대외비/비밀) 구분 필수.  
  **느낀 점**: "청크에 기밀 등급 설정 불가. 파일명에 [대외비] 붙이는 수준. 시스템 접근 제어 필요."
- **청크 승인**: 선임 2명 교차 검토, 전자서명·승인 사유 필수.  
  **느낀 점**: "승인 버튼만 있고 사유·전자서명 없음. 규제 감사 시 책임 입증 불가."
- **청크 관리**: 기밀 청크 검색 제외, 접근 로그 필수.  
  **느낀 점**: "모든 청크 노출. 기밀은 권한자만 열람, 열람 시 로그 필요."

---

## 사용 불편점

| 불편점 | 심각도 | 영향 |
|--------|--------|------|
| 기밀 등급 설정 불가 | 최상 | 보안 규정 위반, 유출 위험 |
| 접근 제어 부재 | 최상 | 무권한자 기밀 열람 가능 |
| 전자서명 미지원 | 상 | 규제 기관 감사 대응 불가 |
| 접근 로그 부재 | 상 | 내부자 위협 대응 불가 |

---

## 변경 요구사항

1. 기밀 등급·접근 제어 (일반/대외비/비밀/1급비밀, RBAC, 검색·RAG 필터링)  
2. 전자서명·승인 사유 (PKI 연동, 사유 10자 이상 필수)  
3. 접근 로그 (열람·수정·삭제 시 사용자·시간·IP, 관리자 조회·내보내기)  
4. 보안 강화 모드 (다운로드 금지, 워터마크)

---

## 업그레이드 시나리오 제안

- **옵션 A**: 생성(기밀 등급·접근 권한) → 승인(전자서명 필수) → 관리(접근 로그)  
- **옵션 B**: 원자력안전법 준수 템플릿, 규제 제출용 자동 분류, 감사 보고서 자동 생성

---

## 기대 효과 (원자력연구원 관점)

| 개선 항목 | 현재 리스크 | 개선 후 |
|-----------|-------------|---------|
| 기밀 유출 | 무권한자 접근 | 접근 제어 100% |
| 규제 감사 | 승인 이력 불충분 | 전자서명+이력 완벽 보존 |
| 내부자 위협 | 열람자 불명 | 접근 로그 실시간 추적 |

원자력안전법 제104조, IAEA Nuclear Security, 산업기술보호법 대응.
