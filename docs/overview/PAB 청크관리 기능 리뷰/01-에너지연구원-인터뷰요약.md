## 현재 기능 사용 경험

- **청크 생성**: 월 1회 100~200페이지 보고서, 헤딩 기준 분할·장 단위 병합.  
  **느낀 점**: "2단계에서 체크박스 200개 확인. '헤딩 레벨 2 자동 병합' 프리셋 있으면 좋겠다."
- **청크 승인**: 전체 승인 위주, AI 추천 거의 미확인.  
  **느낀 점**: "승인 전 일괄 라벨 적용 옵션이 있으면 나중에 검색이 쉬울 것 같다."
- **청크 관리**: 분기 1회 재분류, 여러 청크에 동일 라벨 추가 번거로움.  
  **느낀 점**: "여러 청크 선택 → 라벨 일괄 추가가 안 된다. 10개 선택 후 한 번에 추가되면 10배 줄어든다."

---

## 사용 불편점

| 불편점 | 심각도 | 영향 |
|--------|--------|------|
| 대용량 문서 분할 시 체크박스 과다 | 상 | 2단계 200개 수동 확인 (20분) |
| 전체 승인 시 AI 추천 무시 | 중 | 검색 품질 저하 |
| 청크 관리 다중 선택 불가 | 상 | 10개 청크에 10번 반복 |
| 3개 메뉴 이동 시 컨텍스트 손실 | 중 | 처리 중 청크 기억 안 남 |
| 프리셋 기능 부재 | 중 | 매번 동일 분할 방식 수동 설정 |

---

## 변경 요구사항

1. 자동 병합 프리셋 (헤딩 레벨 2, 100줄 단위 등) + 프로젝트별 프리셋 저장  
2. 일괄 라벨 적용 (승인 전 전체 청크에 라벨 적용 옵션, 관리에서 다중 선택 + 일괄 추가)  
3. 워크플로우 단축 (생성→승인 한 화면 "빠른 승인" 또는 "등록과 동시에 승인" 옵션)

---

## 업그레이드 시나리오 제안

- **옵션 A (권장)**: 청크 관리 통합 메뉴 — 탭1 생성, 탭2 승인 대기, 탭3 관리. 등록 후 자동 탭2 이동.  
- **옵션 B**: 생성 2단계 AI 추천 미리보기, 3단계 "등록+승인+라벨 적용" 한 번에. 전문가/빠른 모드 선택.
