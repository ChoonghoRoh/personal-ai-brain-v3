# 4.3 정책연구원 연구본부장 (이정책 박사) — 인터뷰 요약

## 프로필

| 항목 | 내용 |
|------|------|
| 조직 | 국가 정책연구원 (경제정책 연구본부) |
| 경력 | 18년 (거시경제, 산업정책) |
| 팀 규모 | 30명 (5개 팀, 팀별 6명) |
| 주요 업무 | 정책 보고서 발간, 부처 협업, 국회 제출 자료 |

---

## 현재 기능 사용 경험

- **청크 생성**: 연 4회 300~500페이지 보고서, 5개 팀 챕터 통합 업로드.  
  **느낀 점**: "어느 청크가 어느 팀 소관인지 구분 안 됨. '담당 팀' 메타데이터 필요."
- **청크 승인**: 본부장 최종 승인. 팀장 선검토·부처 검토 의견 필요.  
  **느낀 점**: "다단계 승인 불가. 본부장이 500페이지 혼자 검토해야 함."
- **청크 관리**: 국회 질의 대응 시 키워드 검색, 청크 묶어 보고서 재생성.  
  **느낀 점**: "선택한 청크 10개를 새 문서로 내보내기 기능 있으면 국회 자료 작성이 빨라진다."

---

## 사용 불편점

| 불편점 | 심각도 | 영향 |
|--------|--------|------|
| 담당자 정보 미지원 | 상 | 다부서 협업 시 책임 소재 불명 |
| 다단계 승인 불가 | 상 | 팀장→본부장→부처 워크플로우 불가 |
| 청크 재조합 기능 부재 | 중 | 국회 자료 수동 복사·붙이기 |
| 버전 관리 부재 | 중 | 부처 검토 반영 시 이전 버전 손실 |

---

## 변경 요구사항

1. 청크 메타데이터 확장 (담당자, 담당 팀, 작성일, 검토자), 생성 3단계에서 입력  
2. 다단계 승인 워크플로우 (팀장 검토 → 본부장 승인 → 외부 협의), 단계별 의견·이력  
3. 청크 재조합 및 내보내기 (다수 선택 → 새 문서 병합, MD/DOCX/PDF)  
4. 버전 관리 (수정 시 이전 버전 백업, Diff)

---

## 업그레이드 시나리오 제안

- **옵션 A**: 생성(메타데이터) → 승인 1~3단계(팀장→본부장→부처) → 관리(버전+재조합)  
- **옵션 B**: "정책 보고서"·"국회 자료" 템플릿 (챕터·담당자·검토자 필드, 재조합+PDF)

---

## 기대 효과 (정책연구원 관점)

| 개선 항목 | 현재 문제 | 개선 후 |
|-----------|-----------|---------|
| 다부서 협업 | 책임 소재 불명 | 담당자 추적 100% |
| 보고서 검토 | 본부장 단독 2주 | 팀장 선검토 → 1주 단축 |
| 국회 자료 | 수동 4시간 | 청크 재조합 30분 |

정부 부처 협업·국회 대응 75% 단축, 정책 의사결정 추적성 확보.
