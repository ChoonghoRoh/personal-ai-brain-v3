### 4.2 화학연구원 수석연구원 (박화학 박사)

**조직**: 국가 화학연구원 (나노소재 연구본부)  
**경력**: 20년 (유기화학, GMP 준수 실험)  
**팀 규모**: 8명 (연구원 5명, 안전관리자 1명, 행정 2명)  
**주요 업무**: 실험 프로토콜 문서화, 안전 데이터 시트(SDS) 관리, 특허 출원

#### 4.2.1 현재 기능 사용 경험

**청크 생성 사용 시나리오**: 실험 프로토콜 문서 (10~30페이지) 주 2~3회 업로드, 문단 기준 분할, 안전 관련 섹션 독립 청크 유지.  
**느낀 점**: "2단계에서 안전 관련 청크를 섞이지 않게 확인하는 게 중요한데, 청크 내용을 일일이 펼쳐봐야 합니다. '특정 키워드 포함 청크 자동 선택/해제' 기능이 있으면 실수를 줄일 수 있습니다."

**청크 승인 사용 시나리오**: 안전관리자 승인, Draft에서 안전 키워드 확인, 거절 시 사유 상세 기록.  
**느낀 점**: "승인·거절 이력이 청크 상세에 남지 않습니다. 누가, 언제, 왜 거절했는지 기록이 남아야 실험실 안전 감사 시 입증 자료로 쓸 수 있습니다."

**청크 관리 사용 시나리오**: 화학물질명, CAS 번호 라벨 추가. AI 키워드 추천이 화학 용어 오인식.  
**느낀 점**: "AI가 'Benzene'을 일반 키워드로 인식합니다. 화학물질 데이터베이스(ChEMBL, PubChem)와 연동해서 검증된 용어만 추천하면 신뢰도가 높아질 것 같습니다."

#### 4.2.2 사용 불편점

| 불편점 | 심각도 | 영향 |
|--------|--------|------|
| **안전 청크 식별 어려움** | 상 | 실험 사고 위험 증가 |
| **승인·거절 이력 미보존** | 상 | GMP/ISO 감사 대응 불가 |
| **AI 키워드 전문 용어 오인식** | 중 | 화학물질 오분류 → 검색 실패 |
| **역할 기반 접근 제어 부재** | 중 | 안전관리자 전용 승인 권한 설정 불가 |

#### 4.2.3 변경 요구사항

1. **안전 청크 자동 태깅**: 키워드 사전 기반 자동 라벨, 승인 시 안전 라벨 청크 경고 표시.  
2. **감사 추적(Audit Trail)**: 승인·거절·라벨 변경 이력 청크에 저장, "수정 이력" 탭.  
3. **외부 DB 연동 AI 추천**: ChEMBL, PubChem API로 화학물질명 검증, 검증된 용어만 추천.  
4. **역할 기반 워크플로우**: 연구원(생성·관리), 안전관리자(승인), 수석연구원(전체).

#### 4.2.4 업그레이드 시나리오 제안

**옵션 A**: 생성 → AI 안전 키워드 자동 태깅 → 승인(안전관리자 전용) → 관리(감사 이력 보존).  
**옵션 B**: "화학연구 모드" — 안전 키워드 사전, 외부 화학 DB 연동, 감사 이력 자동 기록.

#### 4.2.5 기대 효과 (화학연구원 관점)

| 개선 항목 | 현재 리스크 | 개선 후 기대 효과 |
|-----------|-------------|------------------|
| 안전 청크 누락 | 실험 사고 위험 | **100% 자동 식별** |
| 감사 추적 부재 | ISO/GMP 인증 실패 | **완전한 이력 관리** |
| 화학물질 오분류 | 실험 데이터 손실 | **전문 용어 정확도 95%+** |

GMP·ISO 17025·국가 화학물질 관리 규정 자동 준수.
