### 4.5 KAIST 교수 (AI Lab) (최카이스트 교수)

**조직**: KAIST 전산학부 AI Lab  
**경력**: 10년 (NLP, Transformer 모델 전문)  
**팀 규모**: 15명 (석사 8명, 박사 5명, 박사후연구원 2명)  
**주요 업무**: 논문 작성, 학생 지도, 산학 협력 프로젝트

#### 4.5.1~4.5.5 요약

- **사용 경험**: 학생별 논문 초고 업로드·섹션 분할·학생명 태깅. 관리자만 생성 접근 → 학생별 개인 공간+교수 승인만 희망. AI 추천 시 '머신러닝'·'기계학습' 중복 추천 → 라벨 정규화 필요. Jupyter 자동 임포트 희망.
- **불편점**: 학생별 권한 분리 불가, AI 라벨 중복 추천, API 연동 부족, 논문 인용 형식 미지원.
- **요구사항**: 학생별 Workspace, 라벨 정규화·동의어 관리, API·Jupyter·Notion 예제, 논문 인용(BibTeX·APA·IEEE).
- **시나리오**: 학생 개인 공간 → 교수 승인 대시보드(라벨 정규화) → 연구실 공유 지식베이스; 또는 arXiv·Hugging Face·Jupyter Extension.
- **기대 효과**: 개인 공간→충돌 0%, 라벨 정규화→검색 +30%, API→자동 임포트(5분→30초). 연구 재현성·논문 작성 시간 단축.

**풀 리뷰 원문**: 원본 문서 §4.5 참조.
