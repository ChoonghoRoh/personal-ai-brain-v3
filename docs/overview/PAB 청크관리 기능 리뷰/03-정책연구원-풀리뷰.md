### 4.3 정책연구원 연구본부장 (이정책 박사)

**조직**: 국가 정책연구원 (경제정책 연구본부)  
**경력**: 18년 (거시경제, 산업정책 전문)  
**팀 규모**: 30명 (5개 팀, 팀별 6명)  
**주요 업무**: 정책 보고서 발간, 부처 협업, 국회 제출 자료 작성

#### 4.3.1 현재 기능 사용 경험

**청크 생성**: 연 4회 대형 정책 보고서 (300~500페이지), 5개 팀 챕터 통합 업로드, 챕터별 담당자 정보 유지 필요.  
**느낀 점**: "5개 팀이 작성한 내용을 통합 업로드하면, 어느 청크가 어느 팀 소관인지 구분이 안 됩니다. 청크에 '담당 팀' 메타데이터를 추가할 수 있으면 좋겠습니다."

**청크 승인**: 본부장 최종 승인, 팀장 5명이 자기 팀 챕터만 선검토, 승인 전 부처 검토 의견 첨부 필요.  
**느낀 점**: "다단계 승인이 안 됩니다. 팀장 검토 → 본부장 승인 → 부처 협의 단계를 거쳐야 하는데, 현재는 본부장이 500페이지를 혼자 검토해야 합니다."

**청크 관리**: 국회 질의 대응 시 키워드로 과거 정책 검색, 관련 청크 묶어 보고서 재생성.  
**느낀 점**: "청크 관리에서 '선택한 청크 10개를 새 문서로 내보내기' 기능이 있으면 국회 제출 자료를 빠르게 만들 수 있습니다."

#### 4.3.2 사용 불편점

담당자 정보 미지원(상), 다단계 승인 불가(상), 청크 재조합 기능 부재(중), 버전 관리 부재(중).

#### 4.3.3 변경 요구사항

1. 청크 메타데이터 확장(담당자, 담당 팀, 작성일, 검토자), 생성 3단계에서 입력.  
2. 다단계 승인 워크플로우(팀장 검토 → 본부장 승인 → 외부 협의), 단계별 의견·이력.  
3. 청크 재조합 및 내보내기(다수 선택 → 새 문서 병합, MD/DOCX/PDF).  
4. 버전 관리(수정 시 이전 버전 백업, Diff).

#### 4.3.4 업그레이드 시나리오 제안

**옵션 A**: 생성(메타데이터) → 승인 1~3단계(팀장→본부장→부처) → 관리(버전+재조합).  
**옵션 B**: "정책 보고서"·"국회 자료" 템플릿.

#### 4.3.5 기대 효과 (정책연구원 관점)

다부서 협업 담당자 추적 100%, 보고서 검토 팀장 선검토로 1주 단축, 국회 자료 4시간→30분. 정부 부처 협업·국회 대응 75% 단축.
