## 프로필

| 항목 | 내용 |
|------|------|
| 조직 | 국가 화학연구원 (나노소재 연구본부) |

---

## 현재 기능 사용 경험

- **청크 생성**: 실험 프로토콜 10~30페이지 주 2~3회, 문단 기준 분할. 안전 섹션 독립 청크 유지.  
  **느낀 점**: "안전 관련 청크 확인 시 일일이 펼쳐봐야 한다. '특정 키워드 포함 청크 자동 선택/해제' 있으면 실수 감소."
- **청크 승인**: 안전관리자 승인, 거절 시 사유 기록 필요.  
  **느낀 점**: "승인·거절 이력이 청크 상세에 남지 않는다. 감사 시 입증 자료 필요."
- **청크 관리**: 화학물질명·CAS 라벨, AI가 화학 용어 오인식.  
  **느낀 점**: "Benzene을 일반 키워드로 인식. ChEMBL·PubChem 연동 검증 용어만 추천하면 신뢰도 상승."

---

## 사용 불편점

| 불편점 | 심각도 | 영향 |
|--------|--------|------|
| 안전 청크 식별 어려움 | 상 | 실험 사고 위험 |
| 승인·거절 이력 미보존 | 상 | GMP/ISO 감사 대응 불가 |
| AI 키워드 전문 용어 오인식 | 중 | 화학물질 오분류 → 검색 실패 |
| 역할 기반 접근 제어 부재 | 중 | 안전관리자 전용 승인 권한 설정 불가 |

---

## 변경 요구사항

1. 안전 청크 자동 태깅 (키워드 사전 기반), 승인 시 안전 라벨 경고  
2. 감사 추적(Audit Trail): 승인·거절·라벨 변경 이력 저장, 수정 이력 탭  
3. 외부 DB 연동 AI 추천 (ChEMBL, PubChem 검증 용어만)  
4. 역할 기반 워크플로우 (연구원/안전관리자/수석연구원 권한 분리)

---

## 업그레이드 시나리오 제안

- **옵션 A**: 생성 → AI 안전 키워드 자동 태깅 → 승인(안전관리자) → 관리(감사 이력 보존)  
- **옵션 B**: "화학연구 모드" — 안전 키워드 사전, 외부 화학 DB 연동, 감사 이력 자동 기록
