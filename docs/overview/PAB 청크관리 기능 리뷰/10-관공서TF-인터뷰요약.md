## 프로필

| 조직 | 정부 부처 디지털 전환 TF팀 |

## 현재 기능 사용 경험

- **청크 생성**: 분기 1회 지침·규정 200~300페이지, 결재선(기안자→과장→국장)·보안 등급 필수.  
  **느낀 점**: "청크에 결재 정보 입력 불가. 문서 관리 규정상 기안·승인 기록 없으면 공문서로 인정 안 된다."
- **청크 승인**: 과장→국장 순 승인, 단계별 의견 필수. 최종 승인 후 국민 서비스 노출.  
  **느낀 점**: "1명만 승인 가능. 과장 선검토→국장 최종 승인 다단계 결재 필요."
- **청크 관리**: 민원 대응 시 규정 청크 검색, 청크 PDF로 공문 첨부.  
  **느낀 점**: "'공식 문서로 내보내기' 없음. 복사·붙이기로 HWP 만드는데 서식 깨진다."

---

## 사용 불편점

| 불편점 | 심각도 | 영향 |
|--------|--------|------|
| 결재선 지원 없음 | 최상 | 공문서 요건 미충족 → 법적 효력 없음 |
| 다단계 승인 불가 | 최상 | 행정 절차 준수 불가 |
| 공식 문서 내보내기 없음 | 상 | HWP 수동 4시간 |
| 보안 등급 미지원 | 상 | 대외비 유출 위험 |

---

## 변경 요구사항

1. 결재선·전자결재 연동 (기안자·검토자·승인자, 정부 전자결재 시스템 연동)  
2. 다단계 승인 (과장 검토 의견 필수 → 국장 승인 서명, 단계별 이력)  
3. 공식 문서 템플릿 (HWP·DOCX·PDF, 정부 표준 서식·표지·결재란·쪽번호)  
4. 보안 등급·열람 로그 (일반/대외비/비밀, 사용자·시간·IP 로그)

---

## 업그레이드 시나리오 제안

- **옵션 A**: 청크 생성+결재선 → 다단계 승인(과장 검토→팀장 수정→과장 재검토→국장 승인) → 공식 문서 발간(HWP 표준 서식·웹사이트 게시)  
- **옵션 B**: 행정안전부 문서 관리 규정 준수 템플릿, 정부24·민원24 연동, 공공기관 보안 가이드라인

## 기대 효과 (공공기관 관점)

| 개선 항목 | 현재 문제 | 개선 후 |
|-----------|-----------|---------|
| 결재 절차 | 수동 HWP+전자결재 별도 4시간 | 통합 워크플로우 30분 |
| 보안 관리 | 대외비 노출 위험 | 보안 등급 자동 적용 → 유출 0% |
| 국민 서비스 | 규정 검색 어려움 | 청크 기반 챗봇 → 즉시 답변 |

정부혁신 31·국민 민원 50% 단축·공공 데이터 개방 품질·행정 투명성.
