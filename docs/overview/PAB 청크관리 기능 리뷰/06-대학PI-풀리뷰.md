### 4.6 대학교수 연구실 PI (한교수 교수)

**조직**: 지방 국립대학교 기계공학과  
**경력**: 8년 (로봇공학, 제어이론)  
**팀 규모**: 5명 (석사 4명, 학부연구생 1명)  
**주요 업무**: 학부 강의, 소규모 과제 수행, 학생 논문 지도

#### 4.6.1~4.6.5 요약

- **사용 경험**: 학기별 강의·과제 보고서. UI 복잡해 학생들이 어려워해 교수가 대신 업로드. "파일 업로드→자동 처리" 버튼 희망. AI 추천 영문→한글 전용 모드 희망. 라벨 자동 부여 희망.
- **불편점**: UI 복잡도, 한글 지원 부족, 자동화 부족, 예산 제약 미고려.
- **요구사항**: 간단 모드(자동 분할·승인·라벨), 한글 우선, 자동 라벨링(신뢰도 0.7 이상), 경량화(텍스트 전용, 10명 미만 프리셋).
- **시나리오**: 파일 업로드→자동 처리→AI 최적 분할·자동 승인·자동 라벨(한글)→완료; 또는 "대학 강의 자료"·"학생 과제" 템플릿.
- **기대 효과**: 사용법 2시간→15분(88%), 청크 생성·승인 3시간→10분(94%), 라벨 관리 자동화. 디지털 격차 해소.

**풀 리뷰 원문**: 원본 문서 §4.6 참조.
