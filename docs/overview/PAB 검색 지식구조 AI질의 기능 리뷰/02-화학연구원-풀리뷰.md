# 2. 화학연구원 (박화학 박사) — 풀 리뷰

**요약본**: [02-화학연구원-인터뷰요약.md](02-화학연구원-인터뷰요약.md)
**모델 주석**: GPT-5.2-Codex 단일 모델 사용(모델 교체 불가)

---

## 1) 현재 기능 설명 및 사용 흐름

### 검색

- 위험 물질/공정 키워드로 문서 탐색
- 안전 라벨/등급 필터는 제한적

### 지식 구조

- 라벨/관계 기반 구조화 가능
- 안전 규정 기반 라벨링은 수동

### AI 질의

- 질의응답은 가능하나 안전 정책 필터가 약함

---

## 2) 사용 후기

- **좋은 점**: 검색 속도와 의미 검색은 유용
- **아쉬운 점**: 안전 관리·감사 대응에 필요한 기록이 부족

---

## 3) 사용 불편점(상세)

| 구분 | 문제                     | 영향                  |
| ---- | ------------------------ | --------------------- |
| 안전 | 안전 라벨 자동 태깅 없음 | 위험 정보 혼입        |
| 감사 | 승인/거절 이력 불완전    | GMP/ISO 감사 지연     |
| 정책 | AI 답변 통제 부족        | 위험한 절차 제안 가능 |

---

## 4) 변경해야 할 사항(우선순위)

1. **안전 키워드 기반 자동 태깅** 및 경고 UI
2. 승인·거절·수정 이력 **감사 로그 통합**
3. AI 질의 **안전 정책/금지어/출처 표시** 적용

---

## 5) 업그레이드/변경/통합 시나리오

- **통합 시나리오 A**: 안전 라벨 필터로 검색 → 지식 구조에서 위험 등급 강조
- **통합 시나리오 B**: AI 질의 시 안전 정책 자동 적용 + 출처 링크 표시

---

## 6) QA/보안/성능 관점

- 안전 라벨 누락 시 알림/차단
- 감사 로그 접근 권한 분리
- AI 질의의 안전 정책 테스트

---

## 7) 테스트 시나리오 제안

### pytest

- 안전 라벨 자동 태깅 규칙 검증
- 승인/거절 감사 로그 저장 검증

### Playwright

- 안전 라벨 필터 검색 흐름
- 위험 키워드 질의 시 정책 차단 확인

---

## 8) 기대 효과

- 안전 감사 대응력 향상
- 위험 지침 생성 가능성 최소화
- 안전 관련 검색 정확도 개선
