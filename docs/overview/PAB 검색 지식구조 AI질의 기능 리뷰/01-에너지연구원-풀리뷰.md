# 1. 에너지연구원 (김에너지 박사) — 풀 리뷰

**요약본**: [01-에너지연구원-인터뷰요약.md](01-에너지연구원-인터뷰요약.md)
**모델 주석**: GPT-5.2-Codex 단일 모델 사용(모델 교체 불가)

---

## 1) 현재 기능 설명 및 사용 흐름

### 검색

- 하이브리드 검색으로 대형 문서에서 핵심 문장 탐색
- 필터는 기본 수준(날짜/라벨/타입 중심)

### 지식 구조

- 라벨/관계 기반 구조화 가능
- 청크 승인/라벨 적용 흐름이 관리자 메뉴에 분리됨

### AI 질의

- 대화형 질의응답 가능
- 출처/근거 표기가 약해 내부 검토에서 재확인 필요

---

## 2) 사용 후기

- **좋은 점**: 검색 속도와 의미 검색 품질이 유용
- **아쉬운 점**: 대량 문서 처리에서 일괄 작업이 부족

---

## 3) 사용 불편점(상세)

| 구분   | 문제                     | 영향                       |
| ------ | ------------------------ | -------------------------- |
| 신뢰   | AI 답변에 출처/근거 미흡 | 정책·기술 보고서 신뢰 저하 |
| 통합   | 검색→지식→AI 흐름이 단절 | 업무 흐름 반복/재입력      |
| 생산성 | 일괄 승인/라벨 부족      | 대형 문서 처리 지연        |

---

## 4) 변경해야 할 사항(우선순위)

1. **검색 결과에 근거/매칭 타입 표시**(키워드/벡터/하이브리드)
2. **공통 필터**(프로젝트/라벨/승인 상태) 전 메뉴 통합
3. **일괄 작업**: 다중 선택 라벨·승인·내보내기
4. 검색 결과 기반 **문서 재조합** 기능

---

## 5) 업그레이드/변경/통합 시나리오

- **통합 시나리오 A**: 검색 결과 선택 → 지식 구조 자동 하이라이트 → AI 질의 컨텍스트 전달
- **통합 시나리오 B**: 검색 결과 일괄 선택 → 문서 재조합(보고서 템플릿) → PDF/DOCX

---

## 6) QA/보안/성능 관점

- 대량 검색 성능 검증(응답 300ms 목표)
- 라벨/승인 상태 기반 접근 제어 확인
- 검색 결과 캐시 적용 여부 확인

---

## 7) 테스트 시나리오 제안

### pytest

- 하이브리드 검색 결과의 필터 정확성
- 일괄 승인/라벨 적용 시 무결성

### Playwright

- 검색 → 지식 구조 → AI 질의 전환 흐름
- 검색 결과 내보내기 성공 여부

---

## 8) 기대 효과

- 대형 보고서 처리 시간 50% 단축
- 검색/AI 답변 신뢰도 향상
- 반복 작업 감소(일괄 처리로 30% 절감)
