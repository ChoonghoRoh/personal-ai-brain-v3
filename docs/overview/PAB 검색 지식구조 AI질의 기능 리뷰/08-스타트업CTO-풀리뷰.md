# 8. 스타트업 CTO (강스타트 CTO) — 풀 리뷰

**요약본**: [08-스타트업CTO-인터뷰요약.md](08-스타트업CTO-인터뷰요약.md)
**모델 주석**: GPT-5.2-Codex 단일 모델 사용(모델 교체 불가)

---

## 1) 현재 기능 설명 및 사용 흐름

### 검색

- 하이브리드 검색 품질은 양호

### 지식 구조

- 라벨/관계 기능은 있으나 API 문서와 SDK 부재

### AI 질의

- `/api/ask` 제공, 운영 지표/Rate Limit 정책 부족

---

## 2) 사용 후기

- **좋은 점**: 기능은 있으나 플랫폼/연동 관점에서 미흡

---

## 3) 사용 불편점(상세)

| 구분   | 문제                      | 영향                  |
| ------ | ------------------------- | --------------------- |
| 연동   | OpenAPI/SDK/CLI 미흡      | 외부 연동 비용 증가   |
| 자동화 | Webhook 부재              | 이벤트 기반 흐름 불가 |
| 운영   | Rate Limit/캐시 정책 약함 | 안정성 저하           |

---

## 4) 변경해야 할 사항(우선순위)

1. **OpenAPI 문서 고도화 + SDK/CLI 제공**
2. **Webhook/이벤트 트리거**
3. **Rate Limit/캐싱 정책 강화**

---

## 5) 업그레이드/변경/통합 시나리오

- **통합 시나리오 A**: 검색/지식/AI 공통 API 패키지 제공
- **통합 시나리오 B**: 이벤트 기반 자동화(Webhook)

---

## 6) QA/보안/성능 관점

- API 성능 지표/대시보드 필요
- Rate Limit 분리(일반/LLM)
- 보안 헤더 및 인증 토큰 회전 정책

---

## 7) 테스트 시나리오 제안

### pytest

- API 응답 표준화 테스트
- Rate Limit/캐시 동작 테스트

### Playwright

- API 키 발급/회전 UI 흐름
- Webhook 이벤트 전달 시뮬레이션

---

## 8) 기대 효과

- 연동 개발 비용 40% 절감
- 운영 안정성 향상
