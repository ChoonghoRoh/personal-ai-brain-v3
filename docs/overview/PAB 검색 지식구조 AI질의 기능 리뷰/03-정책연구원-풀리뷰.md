# 3. 정책연구원 (이정책 박사) — 풀 리뷰

**요약본**: [03-정책연구원-인터뷰요약.md](03-정책연구원-인터뷰요약.md)
**모델 주석**: GPT-5.2-Codex 단일 모델 사용(모델 교체 불가)

---

## 1) 현재 기능 설명 및 사용 흐름

### 검색

- 정책 관련 키워드 검색으로 과거 보고서/청크 탐색

### 지식 구조

- 라벨/관계로 정책 맥락 구조화 가능
- 승인·버전·결재 이력이 통합되어 있지 않음

### AI 질의

- 요약/초안 작성에 유용하지만 출처 표기 제한

---

## 2) 사용 후기

- **좋은 점**: 검색 속도와 관련 문서 접근성이 좋음
- **아쉬운 점**: 행정 절차(결재/버전/근거) 요구 미충족

---

## 3) 사용 불편점(상세)

| 구분 | 문제                           | 영향                  |
| ---- | ------------------------------ | --------------------- |
| 근거 | 출처·버전 관리 부족            | 정책 근거 추적 어려움 |
| 절차 | 다단계 승인/결재 부재          | 행정 요구 미충족      |
| 산출 | HWP 등 공식 서식 내보내기 부족 | 업무 지연             |

---

## 4) 변경해야 할 사항(우선순위)

1. **출처/버전 관리 강화** (근거 링크, 변경 이력)
2. **결재/승인 워크플로우** 도입
3. **공식 서식 내보내기** (HWP/DOCX/PDF)

---

## 5) 업그레이드/변경/통합 시나리오

- **통합 시나리오 A**: 검색 결과 → 지식 구조 근거 연결 → AI 질의 요약
- **통합 시나리오 B**: 정책 보고서 템플릿 → 재조합/내보내기 자동화

---

## 6) QA/보안/성능 관점

- 근거 링크/버전 이력 무결성 검증
- 승인/결재 로그 감사 내보내기
- AI 답변 출처 표시 테스트

---

## 7) 테스트 시나리오 제안

### pytest

- 버전 관리/변경 이력 저장 검증
- 승인 단계별 권한 테스트

### Playwright

- 정책 템플릿 내보내기 흐름
- 검색→AI 질의 근거 표시 확인

---

## 8) 기대 효과

- 정책 근거 추적성 100%
- 결재 처리 시간 40% 단축
- 국회 대응 시간 70% 단축
