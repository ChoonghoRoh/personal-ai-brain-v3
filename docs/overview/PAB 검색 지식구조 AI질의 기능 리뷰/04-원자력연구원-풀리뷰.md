# 4. 원자력연구원 (정원자 박사) — 풀 리뷰

**요약본**: [04-원자력연구원-인터뷰요약.md](04-원자력연구원-인터뷰요약.md)
**모델 주석**: GPT-5.2-Codex 단일 모델 사용(모델 교체 불가)

---

## 1) 현재 기능 설명 및 사용 흐름

### 검색

- 의미/키워드 검색 가능
- 기밀 등급 기반 필터링 부재

### 지식 구조

- 라벨/관계 기반 구조화는 가능
- 기밀 등급/권한 분리 미흡

### AI 질의

- 질문 응답은 빠르나 기밀 차단 정책 부족

---

## 2) 사용 후기

- **좋은 점**: 로컬 설치형이라 외부 유출 위험이 상대적으로 낮음
- **아쉬운 점**: 등급 기반 제어 및 감사 로그가 부족

---

## 3) 사용 불편점(상세)

| 구분 | 문제                   | 영향                |
| ---- | ---------------------- | ------------------- |
| 보안 | 등급 필터 부재         | 기밀 유출 위험      |
| 감사 | 열람/수정 로그 미흡    | 규제 감사 대응 불가 |
| 정책 | AI 질의 차단 규칙 미흡 | 민감 정보 노출 가능 |

---

## 4) 변경해야 할 사항(우선순위)

1. **기밀 등급 + RBAC** 적용(검색/지식/AI 공통)
2. **감사 로그 내보내기** 및 관리자 조회
3. **AI 질의 민감 정보 차단**(정책 세트 강화)

---

## 5) 업그레이드/변경/통합 시나리오

- **통합 시나리오 A**: 등급 필터 기반 검색 → 지식 구조 표시 제한
- **통합 시나리오 B**: AI 질의 컨텍스트에서 기밀 청크 자동 제외

---

## 6) QA/보안/성능 관점

- 등급별 접근 차단 테스트
- 로그 무결성/내보내기 기능 검증
- AI 질의 정책 테스트(금지어/민감도)

---

## 7) 테스트 시나리오 제안

### pytest

- RBAC 적용 시 검색 결과 필터링 정확성
- 감사 로그 저장/조회 검증

### Playwright

- 권한별 메뉴 접근 제어
- 기밀 청크 AI 질의 차단 확인

---

## 8) 기대 효과

- 보안 사고 예방, 감사 대응 강화
- 기밀 정보 노출 가능성 최소화
