# 6. 대학 PI (한교수 교수) — 풀 리뷰

**요약본**: [06-대학PI-인터뷰요약.md](06-대학PI-인터뷰요약.md)
**모델 주석**: GPT-5.2-Codex 단일 모델 사용(모델 교체 불가)

---

## 1) 현재 기능 설명 및 사용 흐름

### 검색

- 기본 검색은 가능하나 필터/설정이 복잡

### 지식 구조

- 라벨/관계 관리 가능
- 설정 부담이 커 초보자에게 장벽

### AI 질의

- 요약/질의 기능은 있으나 가이드 부족

---

## 2) 사용 후기

- **좋은 점**: 기능은 풍부하나 초보자 중심 UX 부족
- **아쉬운 점**: 교육/연구 병행 환경에 맞는 간소화 부족

---

## 3) 사용 불편점(상세)

| 구분 | 문제                     | 영향             |
| ---- | ------------------------ | ---------------- |
| UX   | 학습 곡선 높음           | 신규 사용자 이탈 |
| 언어 | 한글 우선/전용 모드 부재 | 학생 이해도 저하 |
| 운영 | 자동화 부족              | 운영 부담 증가   |

---

## 4) 변경해야 할 사항(우선순위)

1. **초보자 모드/온보딩 가이드**
2. **한글 우선 검색/AI 질의**
3. **업로드→분할→승인 자동화**

---

## 5) 업그레이드/변경/통합 시나리오

- **통합 시나리오 A**: 1버튼 업로드 → 자동 라벨 → 검색/AI 요약
- **통합 시나리오 B**: 강의/과제 템플릿 기반 재조합/내보내기

---

## 6) QA/보안/성능 관점

- 초보자 모드 UX 테스트
- 한글 검색 품질 테스트
- 자동화 작업 오류/롤백 검증

---

## 7) 테스트 시나리오 제안

### pytest

- 한글 형태소 기반 검색 정확도
- 자동 라벨링 안정성

### Playwright

- 초보자 모드 온보딩 흐름
- 템플릿 내보내기

---

## 8) 기대 효과

- 교육 자료 준비 시간 40% 단축
- 학생 사용성 개선
