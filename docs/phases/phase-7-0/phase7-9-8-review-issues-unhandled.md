# Phase 7.9.8: 처리 안된 부분 정리

**작성일**: 2026-01-10

---

## 🔴 처리 안된 부분 목록

### 1. 페이징 기능 없음

**영향 받는 페이지**:

1. `/knowledge` - 청크 목록
2. `/admin/approval` - 청크 목록
3. `/logs` - 로그 목록
4. `/admin/labels` - 청크 검색 결과

**문제점**:

- 모든 데이터를 한 번에 로드하여 성능 문제 가능성
- 많은 데이터가 있을 경우 로딩 시간 증가
- 메모리 사용량 증가

**현재 상태**:

- 일부 페이지는 `limit` 파라미터로 제한하지만 UI에 페이징 없음
- 사용자가 더 많은 데이터를 볼 수 없음

**해결 방안**:

1. 백엔드: `offset`, `limit` 파라미터 지원
2. 프론트엔드: 페이지 번호 또는 무한 스크롤 구현
3. 페이지당 항목 수 선택 옵션

**우선순위**: 🟡 중간

---

### 2. 검색 기능 부족

**영향 받는 페이지**:

1. `/knowledge` - 청크 내용 검색 없음
2. `/admin/labels` - 청크 검색은 있으나 고급 검색 옵션 없음
3. `/logs` - 기본 검색만 있음

**문제점**:

- 필터링 옵션이 제한적
- 정규식 검색 불가
- 복합 조건 검색 불가

**해결 방안**:

1. 고급 검색 옵션 추가
2. 정규식 검색 지원
3. 복합 조건 검색 (AND, OR)

**우선순위**: 🟢 낮음

---

### 3. 정렬 옵션 없음

**영향 받는 페이지**:

1. `/knowledge` - 청크 정렬 옵션 없음
2. `/admin/approval` - 청크 정렬 옵션 없음
3. `/logs` - 로그 정렬 옵션 없음
4. `/search` - 검색 결과 정렬 옵션 없음

**문제점**:

- 날짜순으로만 정렬
- 사용자가 원하는 순서로 정렬 불가

**해결 방안**:

1. 날짜순/역순 정렬
2. 이름순 정렬
3. 관계 수순 정렬 (지식 구조)
4. 유사도순 정렬 (검색)

**우선순위**: 🟢 낮음

---

### 4. 대화 기록 영구 저장 없음

**영향 받는 페이지**:

1. `/ask` - AI 질의 대화 기록

**문제점**:

- 대화 기록이 메모리에만 저장
- 페이지 새로고침 시 사라짐
- 대화 기록을 나중에 확인할 수 없음

**해결 방안**:

1. 로컬 스토리지에 저장
2. 또는 서버에 저장 (선택적)
3. 대화 기록 내보내기 기능

**우선순위**: 🟡 중간

---

### 5. 일괄 작업 기능 없음

**영향 받는 페이지**:

1. `/admin/approval` - 일괄 승인/거절 없음
2. `/admin/labels` - 여러 청크에 동시 라벨 추가 없음
3. `/admin/groups` - 키워드 일괄 연결은 있으나 개선 필요

**문제점**:

- 개별 작업만 가능하여 시간 소모
- 많은 항목을 처리할 때 비효율적

**해결 방안**:

1. 체크박스 선택 기능
2. 선택한 항목 일괄 처리
3. 진행 상태 표시

**우선순위**: 🟢 낮음

---

### 6. 거절 사유 입력 없음

**영향 받는 페이지**:

1. `/admin/approval` - 청크 거절

**문제점**:

- 청크 거절 시 사유를 입력할 수 없음
- 나중에 왜 거절했는지 확인하기 어려움
- 거절 사유 통계 불가

**해결 방안**:

1. 거절 시 사유 입력 모달
2. 거절 사유 저장
3. 거절 사유 통계

**우선순위**: 🟢 낮음

---

### 7. 라벨 삭제 전 확인 없음

**영향 받는 페이지**:

1. `/admin/labels` - 라벨 삭제
2. `/admin/groups` - 그룹 삭제

**문제점**:

- 삭제 전 확인 다이얼로그 없음
- 실수로 중요한 라벨/그룹을 삭제할 수 있음
- 삭제 전 영향도 확인 없음

**해결 방안**:

1. 삭제 전 확인 다이얼로그
2. 삭제 전 영향도 표시 (해당 라벨이 붙은 청크 수 등)
3. 삭제 취소 불가능 경고

**우선순위**: 🟡 중간

---

### 8. 로그 내보내기 없음

**영향 받는 페이지**:

1. `/logs` - 작업 로그

**문제점**:

- 로그를 파일로 내보낼 수 없음
- 로그 분석을 위해 외부 도구 사용 불가

**해결 방안**:

1. CSV 형태로 내보내기
2. JSON 형태로 내보내기
3. Markdown 형태로 내보내기

**우선순위**: 🟢 낮음

---

### 9. 답변 스트리밍 없음

**영향 받는 페이지**:

1. `/ask` - AI 질의
2. `/reason` - Reasoning

**문제점**:

- 답변이 완전히 생성된 후에만 표시됨
- 긴 답변의 경우 사용자 대기 시간 증가
- 진행 상태를 알 수 없음

**해결 방안**:

1. 스트리밍 응답 지원
2. 실시간으로 답변 표시
3. 진행 상태 표시

**우선순위**: 🟢 낮음

---

### 10. 결과 저장/공유 없음

**영향 받는 페이지**:

1. `/reason` - Reasoning 결과
2. `/ask` - AI 질의 답변

**문제점**:

- Reasoning 결과나 AI 답변을 저장할 수 없음
- 결과를 공유할 수 없음
- 나중에 결과를 다시 확인하기 어려움

**해결 방안**:

1. 결과 저장 기능
2. 결과 공유 기능 (링크 생성)
3. 결과 히스토리

**우선순위**: 🟢 낮음
