# Phase 7.7 기능 테스트 범위

⸻

1️⃣ Keyword Group Management (키워드 그룹 관리)

접근 경로
http://localhost:8000/knowledge-admin

확인 방법 1. Knowledge Admin 페이지 접근 2. 상단 탭에서 “📦 키워드 그룹” 클릭

확인 항목
• 좌측: 키워드 그룹 카드 리스트
→ 그룹 이름 / 설명 / 색상 / 키워드 수 표시
• 우측: 키워드 카드 리스트
→ 키워드 이름 / 속한 그룹 표시
• 상단: 검색 입력창 및 매칭 모드 토글 확인
• 하단: 선택 요약 바 확인
→ 선택된 그룹 / 키워드 수, 일괄 적용 버튼 표시
• 그룹 생성 버튼 클릭 → 모달에서
“💡 설명 기반 키워드 추천” 버튼 존재 확인

⸻

2️⃣ 라벨 매칭 카드

접근 경로
http://localhost:8000/knowledge

확인 방법 1. Knowledge Studio 페이지 접근 2. 청크 카드 클릭하여 상세 정보 열기 3. “💡 라벨 매칭” 탭 기본 활성화 확인

확인 항목
• 상단 탭:
→ “추천 키워드” / “추천 그룹” 전환 가능
• 추천 라벨 카드 목록 표시
→ 이름 / 타입 / 신뢰도 / [추가] 버튼 존재
• 이미 연결된 라벨
→ “연결됨” 상태 비활성화 처리 확인
• “연결된 라벨” 영역에서
→ 라벨 칩 “×” 버튼으로 제거 가능 여부 확인

⸻

3️⃣ 관계 매칭 보드

접근 경로
http://localhost:8000/knowledge

확인 방법 1. Knowledge Studio 페이지 접근 2. 청크 카드 클릭하여 상세 정보 열기 3. “🔗 관계 매칭” 탭 클릭

확인 항목
• 3단 레이아웃 구성 확인
• 좌측: 기준 청크 카드 (내용 미리보기 + 메타데이터)
• 중앙: 연결된 관계 카드 리스트
• 관계 타입 배지
• 확정 / 제안 상태 배지
• [해제] 버튼
• 우측: AI 추천 관계 카드 리스트
• 관계 타입 배지
• 유사도 점수 & 막대 그래프
• [연결] / [무시] 버튼
• 하단: 선택된 관계 요약 바
→ 다중 선택 시 표시됨
• 기능 확인
• 카드 클릭 → 다중 선택 가능
• [선택한 관계 연결] 버튼 → 일괄 연결 확인

⸻

4️⃣ Reasoning 필터 확장

접근 경로
http://localhost:8000/reason

확인 방법 1. Reasoning Lab 페이지 접근 2. 브라우저 개발자 도구 (F12) 실행 → Network 탭 열기 3. Reasoning 실행 시 POST /api/reason 요청 확인

Request Body filters 구조 확인

{
"filters": {
"project_ids": [...],
"category_label_ids": [...],
"keyword_group_ids": [...],
"keyword_ids": [...]
}
}

또는 API 문서에서 확인
http://localhost:8000/docs
/api/reason 엔드포인트 → Try it out → filters 추가 후 테스트

# Phase 7.7 기능 테스트 항목별 상세 계획

## 1. 목적

Phase 7.7 기능을 대상으로 1차 코드 테스트(backend, front, DB, brain, docs 연계)와 2차 사용자 UI 테스트를 단계적으로 수행하여 품질을 확보한다.

## 2. 대상 기능

1️⃣ Keyword Group Management (키워드 그룹 관리)
2️⃣ 라벨 매칭 카드
3️⃣ 관계 매칭 보드
4️⃣ Reasoning 필터 확장

## 3. 대상 기능 별 테스트 진행 방식

1단계: 1차 코드 테스트

- backend API, 프론트 UI 상태, DB 스키마·데이터 반영, brain 추천 로직, docs 연계까지 포함한 단위 수준 통합 점검
- 개발 코드, 변수, 타입, API 스펙, 데이터 흐름 기준 확인

2단계: 2차 UI 사용자 테스트

- 실제 사용 시나리오 기반
- 사용성, 흐름, 이해도, 직관성 중심 확인
- 결과는 체크리스트 기반 개선 반복

---

- 그룹 생성 UX
- 설명 기반 추천 UX
- 매칭 모드 UX
- 검색 UX
- 피드백 반영 구조

### 4.2 라벨 매칭 카드

#### 1차 코드 테스트 체크리스트

Backend, Frontend, DB, Brain, Docs 항목 포함

## 테스트 진행 방식

- 4가지 대상 기능 별 step by step으로 진행
- 기능별 1단계 수행 완료 후 2단계 사용자 테스트 개선 방식으로 진행
- 단계별 세부 테스트 기록 및 개선 사항 기록

## 테스트 완료 기준

- 1차 코드 테스트: 모든 항목 Pass 또는 Defect 등록
- 2차 사용자 테스트: 치명적 UX 문제 제거 및 사용성 개선, 데이터 흐름 개선
