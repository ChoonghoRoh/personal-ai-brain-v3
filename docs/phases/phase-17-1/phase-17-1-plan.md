# Phase 17-1 Plan — 키워드 그룹 3단 레이아웃 리뉴얼

**Phase**: 17-1
**목표**: 2단(목록+모달) → 3단(목록+상세+키워드) 레이아웃으로 전환
**선행**: Phase 16 완료
**기준 문서**: [Phase 17 개발 요구사항 §2](../../planning/260218-0830-phase17-개발-요구사항.md)

---

## 1. 배경

키워드 그룹 관리 페이지가 2단 레이아웃(목록+모달)으로 운영 중이나, 다음 문제가 있다:
- 그룹 클릭 시 모달이 열려 전체 흐름이 끊김
- 키워드 목록이 모달 내부에 있어 동시 확인 불가
- 수정/삭제 버튼이 목록 내부에 위치하여 조작 혼란
- 페이지네이션 없이 전체 로드 → 대량 그룹 시 성능 저하

## 2. 범위

| Task | 도메인 | 내용 | 담당 |
|------|--------|------|------|
| 17-1-1 | [BE] | 그룹 목록 페이지네이션 API (page/size + total) | backend-dev |
| 17-1-2 | [FE] | 3단 레이아웃 HTML 구조 (groups.html 전면 개편, 모달 제거) | frontend-dev |
| 17-1-3 | [FE] | 상세 패널 인라인 편집 (crud.js 모달→인라인 전환) | frontend-dev |
| 17-1-4 | [FE] | 키워드 목록 3단 영역 이동 (matching.js) | frontend-dev |
| 17-1-5 | [FE] | 페이지네이션 + 자동 선택 (ui.js) | frontend-dev |
| 17-1-6 | [FE] | CSS 3단 그리드 (admin-groups.css) | frontend-dev |
