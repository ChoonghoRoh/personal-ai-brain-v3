# Phase 18 Master Plan: 키워드 그룹 UX 고도화 + 검색 품질 + 지식 구조 확장

> **상태**: DRAFT (검수 후 확정 예정)
> **작성일**: 2026-02-20
> **선행 조건**: Phase 17 체인 완료 (17-1 ~ 17-8)

---

## 목표

1. **키워드 그룹 트리 구조 별도 탭 + 키워드 추천 UX 복구/개선** — 트리 연결 관리 전용 탭 + LLM 키워드 추천 복구 + 하단 액션바 통합
2. **지식 구조를 계층적 트리로 확장** — 문서·청크까지 포괄하는 통합 지식 트리 구조 도입
3. **검색 품질 개선** — 키워드/시맨틱/하이브리드 검색 간 일관성 확보 + 결과 차별화 UX
4. **관계 매칭 고도화** — 추천 관계 알고리즘을 Qdrant 벡터 유사도 기반으로 고도화

---

## 현재 상태 (AS-IS)

### 지식 구조
```
Project (프로젝트)         ← 플랫 목록, 계층 없음
 └── Document (문서)       ← 플랫 목록, 폴더 경로만 존재
      └── Chunk (청크)     ← 문서 내 순서(chunk_index)만
           ├── Label (라벨)  ← ✅ 트리 구현 완료 (Phase 17-8)
           └── Relation (관계) ← 플랫 엣지 목록
```

### 검색
- 키워드 검색: PostgreSQL ILIKE + `approved`/`draft` 필터
- 시맨틱 검색: Qdrant 벡터 유사도 (status 필터 없음)
- 하이브리드: RRF 융합
- **문제**: 모드 간 결과 차이가 UI에서 구분되지 않음

### 관계 매칭
- 추천 관계: Qdrant 벡터 유사도 1차 → DB fallback 2차 (핫픽스 적용)
- **문제**: 추천 품질·정밀도 개선 여지 있음

### 키워드 그룹 관리 (`/admin/groups`)
- Phase 17-8에서 트리 조회/이동/D&D 구현 완료
- **문제 1**: 트리 구조 연결이 기존 리스트뷰에 토글로 붙어 있어 UX 혼재
- **문제 2**: 키워드 추천 LLM 버튼이 삭제됨 → 복구 필요
- **문제 3**: 키워드 상세 수정/삭제/저장 액션이 분산되어 있음 → 하단 액션바 통합 필요

## 기술 검토 사항

### 키워드 그룹 트리 탭 + 추천 UX (18-1)
- 기존 `groups.html` 리스트/트리 토글 → 탭 UI로 전환 (목록 탭 / 트리 연결 탭)
- 트리 연결 탭: 메뉴 관리와 동일한 부모-자식 D&D 방식 (기존 `keyword-group-treeview.js` 재활용)
- LLM 키워드 추천: `handle_suggest_keywords` (suggestions.py) 복구 + 버튼 위치 이동
- 연관 키워드 조회: 동일 키워드 ILIKE + Qdrant 벡터 유사도 결합 → 상위 20개 반환 API 신규
- 하단 액션바: 키워드 선택 시 수정/삭제/저장 버튼이 하단에 고정 표시되는 sticky bar 패턴

### 폴더 트리 구현 방식 (18-2)

- **방안 A**: `file_path` 파싱 → 가상 폴더 트리 (DB 변경 없음)
- **방안 B**: `Folder` 테이블 신규 생성 → 실제 폴더 엔티티 관리
- **권장**: 방안 A 우선 적용 후 필요시 B로 전환

### 검색 일관성 (18-4)
- Qdrant filter에 `status` 필드 추가 필요 (payload 필터)
- 현재 Qdrant에 status가 payload로 저장되어 있는지 확인 필요
- 없으면 임베딩 재생성 시 status 포함하도록 파이프라인 수정

### 관계 추천 (18-5)
- `qdrant_client.search(vector=chunk_vector)` 직접 호출 방식
- cross-document 검색으로 범위 확장
- LLM 기반 관계 타입 자동 분류 (similar vs prerequisite vs extends)
