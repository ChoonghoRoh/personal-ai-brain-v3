## 목표

CSS Grid 레이아웃 변경에 맞춰 HTML 마크업을 정리하고, JS 렌더링이 정상 호환되는지 확인.

## 변경 사항

### 1. HTML 마크업 정리

- 현재 `section.stats-tier` > `section.summary-cards/charts-row/...` 중첩 구조 확인
- 불필요한 중첩 `section` 정리 (CSS Grid에 맞게)
- 시맨틱 태그 적절성 확인
- **주의**: `id` 속성 변경 금지 (JS에서 참조)

### 2. JS 렌더링 호환 확인

**statistics.js**:
- `updateSummaryCards()`: `document.getElementById()` 기반 — 마크업 id 유지 시 호환
- `updateTopLabelsTable()`: `topLabelsBody` tbody 참조 — 호환
- `updateProjectsTable()`: `projectsBody` tbody 참조 — 호환
- `renderFilteredTable()`: `filteredTableHead/Body` 참조 — 호환
- `renderFilteredPagination()`: `filteredPagination` 참조 — 호환

**statistics-charts.js**:
- `canvas` 요소 id 4개: `docTypeChart`, `chunkStatusChart`, `labelTypeChart`, `trendsChart`
- Chart.js `responsive: true`, `maintainAspectRatio: false` 설정 유지
- Grid 변경 시 차트 리사이즈 자동 처리 확인

### 3. 변경 최소화 원칙

- JS 파일은 **호환성 확인**이 주 목적, 불필요한 수정 지양
- HTML 마크업도 CSS Grid 호환을 위한 최소한의 정리만 수행
- `id`, `class`, `data-*` 속성 중 JS에서 참조하는 것은 절대 변경 금지
