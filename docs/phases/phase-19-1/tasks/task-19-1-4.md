## §1 목적

프론트엔드에서 백엔드가 이미 정제한 `new_keywords`를 2차 정제하여 유효 키워드가 제거되는 문제를 해결한다.

| 파일 | 변경 유형 |

| `web/public/js/admin/keyword-group-suggestion.js` L82 | 수정 — `extractKeywordsOnly()` 호출 제거 |

## §3 구현 상세

**현재 문제**: `new_keywords`가 백엔드 `postprocess_korean_keywords()`로 1차 정제 완료 → 프론트에서 `extractKeywordsOnly()` + `cleanKeyword()`로 2차 정제 → 과잉 필터링

**수정**:
```javascript
// 기존 (제거 대상)
const cleaned = extractKeywordsOnly(response.new_keywords);

// 수정 (최소 처리만)
const cleaned = response.new_keywords
    .map(kw => kw.trim())
    .filter(kw => kw.length >= 2);
```

**주의**: `extractKeywordsOnly()`, `cleanKeyword()` 함수 자체는 삭제하지 않음 (다른 곳에서 사용 가능성 + 향후 활용)
