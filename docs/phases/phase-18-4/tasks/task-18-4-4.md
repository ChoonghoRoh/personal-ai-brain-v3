## §1 목적
Qdrant 벡터 유사도 기반 cross-document 관계 추천 API.

## §2 변경 범위
- `backend/routers/knowledge/relations.py` — 추천 API 엔드포인트 추가

## §3 API 스펙
```
GET /api/relations/recommendations?chunk_id={id}&top_k=5&cross_document_only=true
```
응답:
```json
{
  "chunk_id": 1,
  "recommendations": [
    {
      "target_chunk_id": 15,
      "target_document_id": 3,
      "target_document_name": "search.py",
      "similarity_score": 0.87,
      "suggested_type": "similar",
      "target_snippet": "..."
    }
  ]
}
```

## §4 구현 상세
1. chunk_id로 Qdrant point 조회 → 벡터 획득
2. 해당 벡터로 Qdrant search → 유사 청크 top_k 반환
3. cross_document_only=true면 같은 document_id 제외
4. 기존 관계가 있는 쌍은 결과에서 표시 (already_related: true)
