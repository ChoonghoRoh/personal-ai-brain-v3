**Phase ID**: 11-4  
**Phase 명**: 통합 테스트  
**Z**: 0 (초기 설계)  
**기준 문서**: [phase-11-master-plan.md](../phase-11-master-plan.md)

## 1. Phase Goal

Phase 11-1~11-3에서 구축한 **DB·API·Admin UI**를 대상으로 **통합 테스트**를 체계화한다.  
**docs/devtest** 폴더에 테스트 가이드를 문서화하고, **Phase 11 각 Task별 세부 기능에 대해 최대 20가지씩** 통합 테스트 시나리오를 작성·실행하며, **Task당 실행 후 코드 오류·미해결/해결 내용을 리포트에 명시**한다.

**운영 매뉴얼·롤백 절차·백업 연동**은 본 Phase Task에서 제외한다(별도 문서·Phase에서 다룰 수 있음).

### 2.1 In Scope

### 2.2 Out of Scope

- 운영 매뉴얼·롤백 절차·백업 연동 문서화(기존 11-4-2 Task 삭제, 별도 관리 가능).
- 멀티 테넌시·A/B 테스트(마스터 플랜과 동일).
- 신규 기능 개발.

## 4. Task별 상세

### 4.1 Task 11-4-1: docs/devtest 폴더 생성 및 통합 테스트 가이드 문서화

| 항목 | 내용 |
|------|------|
| **목적** | 통합 테스트 수행 방법·환경·산출물 위치를 한곳에 정리. |
| **작업** | `docs/devtest/` 폴더 생성. 통합 테스트 가이드 문서 작성(목적, 환경, 시나리오 작성 규칙, 리포트 규칙, Phase 11 연계). |
| **산출물** | `docs/devtest/README.md`(또는 `docs/devtest/integration-test-guide.md`). |

### 4.2 Task 11-4-2: Phase 11 기능별 통합 테스트 시나리오(Task당 최대 20가지)·디테일 테스트

| 항목 | 내용 |
|------|------|
| **목적** | Phase 11-1, 11-2, 11-3 각 Task의 세부 기능에 대해 통합 테스트 시나리오를 **Task당 최대 20가지** 작성하고, 디테일한 수준까지 테스트를 진행. |
| **대상 Task** | **11-1**: 11-1-1, 11-1-2, 11-1-3. **11-2**: 11-2-1, 11-2-2, 11-2-3, 11-2-4, 11-2-5. **11-3**: 11-3-1, 11-3-2, 11-3-3, 11-3-4. (각 Task당 최대 20개 시나리오.) |
| **작업** | 각 Phase·Task별 시나리오 문서 작성(예: `docs/devtest/phase-11-1-scenarios.md`, `phase-11-2-scenarios.md`, `phase-11-3-scenarios.md` 또는 Task 단위 파일). 시나리오별 전제조건·단계·기대결과·실제결과 기록. 디테일 테스트 실행. |
| **산출물** | `docs/devtest/` 하위 시나리오 문서, 실행 기록. |

### 4.3 Task 11-4-3: Task당 실행 결과 리포트 규정 명시 및 리포트 작성

| 항목 | 내용 |
|------|------|
| **목적** | Task당 실행 후 **코드 오류·미해결 이슈·해결된 이슈**를 리포트에 명시하고, 동일 규칙으로 리포트를 작성. |
| **규정 명시** | 테스트 가이드 또는 별도 문서에 다음을 명시: (1) Task당 실행 결과 리포트에 포함할 항목: 코드 오류(재현 조건·로그·스택), 미해결 이슈(설명·우선순위), 해결된 이슈(해결 방법·커밋/PR 참조). (2) 리포트 저장 위치: `docs/devtest/reports/` 또는 `docs/phases/phase-11-4/` 하위. (3) 파일 명명: 예 `phase11-4-2-task-11-1-1-execution-report.md`. |
| **작업** | 11-4-2 실행 결과를 바탕으로, Phase 11 각 Task(11-1-1~11-3-4)별 실행 결과 리포트 작성. 코드 오류·미해결/해결 내용을 위 규정에 따라 기재. |
| **산출물** | 리포트 규정 문서(가이드에 포함 또는 `docs/devtest/report-format.md`). Task별 실행 결과 리포트(`docs/devtest/reports/` 또는 `docs/phases/phase-11-4/`). |
