# UI·UX 디자이너 관점 — 테스트 수행 가이드

**역할**: 시인성·일관성·접근성·사용성·시각적 계층 검증

---

## 1. 테스트 중 확인할 것

- **버튼/링크 식별 용이성**: 클릭 가능한 요소가 시각적으로 구분되는지, 라벨이 명확한지
- **레이아웃·정렬·여백 일관성**: 화면 간·섹션 간 정렬·간격·폰트 크기가 일관된지
- **포커스·키보드·라벨(접근성)**: 키보드만으로 조작 가능한지, 포커스 표시가 보이는지, 필드에 라벨/placeholder가 적절한지
- **클릭/입력 흐름**: 사용자가 “다음에 무엇을 해야 하는지” 자연스럽게 인지할 수 있는지
- **오류/안내 메시지 가독성**: 에러·성공·안내 문구가 눈에 띄고 읽기 쉬운지

---

## 2. 기록 형식

| 항목        | 내용                                        |
| ----------- | ------------------------------------------- |
| 시나리오 ID | 체크리스트 번호 (예: W4.2)                  |
| 관점        | UIUX                                        |
| 화면/요소   | 문제가 있는 화면 또는 UI 요소               |
| 이슈 설명   | 시인성·일관성·접근성·사용성 관점에서의 이슈 |
| 개선 제안   | 구체적인 개선 방향 (선택)                   |

---

## 3. 실행용 프롬프트 (복사하여 사용)

**프롬프트 A — 체크리스트 수행 안내**

```
나는 UI·UX 디자이너 관점으로 아래 웹 사용자 체크리스트를 수행합니다.
- 각 시나리오를 브라우저에서 실행하면서 다음을 확인합니다:
  1) 버튼/링크가 시각적으로 구분되고 라벨이 명확한지
  2) 레이아웃·정렬·여백이 화면 간 일관된지
  3) 키보드·포커스·라벨 등 접근성이 적절한지
  4) 클릭/입력 흐름이 자연스러운지
  5) 오류/안내 메시지가 가독성 있게 표시되는지
- 발견한 내용은 "시나리오 ID, 관점(UIUX), 화면/요소, 이슈 설명, 개선 제안" 형식으로 기록합니다.
```

**프롬프트 B — 결과 정리용**

```
UI·UX 디자이너 관점 테스트 결과를 정리합니다.
- 시나리오별로: 시인성·일관성·접근성·사용성 관점에서의 이슈와 개선 제안을 한 줄씩 요약해 주세요.
```
